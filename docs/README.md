# docs/ — 真実の源泉

このディレクトリはエージェントが開発を進める際に参照する記録システムだ。
説明資料ではなく、要件・設計・判断根拠の一次情報源として管理する。

---

## ディレクトリ構造

```
docs/
├── decisions/
│   └── YYYY-MM-DD-<topic>.md   # 意思決定ログ
├── plans/
│   └── <issue-number>-<title>.md  # 実行計画
└── <issue-number>-design.md    # 設計エージェントが生成する個別設計書
```

---

## 意思決定ログのフォーマット（`decisions/YYYY-MM-DD-<topic>.md`）

```markdown
# <決定事項のタイトル>

**日時**：YYYY-MM-DD
**決定者**：<人間 or エージェント名>

## 背景
## 選択肢
## 決定内容
## 理由
## 影響範囲
```

---

## 実行計画のフォーマット（`plans/<issue-number>-<title>.md`）

```markdown
# 実行計画：<Issue タイトル>

**対象 Issue**：#<番号>
**作成日**：YYYY-MM-DD

## 目的
## 対象範囲
## 前提条件
## 実装方針
## タスク一覧
- [ ] タスク1
## 検証方法
## リスク・懸念事項
```

---

## 個別設計書のフォーマット（`<issue-number>-design.md`）

設計エージェントが自動生成する。手動作成も可。

```markdown
# 設計書：<Issue タイトル>

**対象 Issue**：#<番号>
**作成日**：YYYY-MM-DD

## 目的・背景
## 実装方針
## 制約条件
## 作業分割方針
## 検証方法
```

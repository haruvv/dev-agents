# docs/ ディレクトリのポリシー

## docs/ に置くもの（共有知識のみ）

- 開発規約・コーディングスタイル
- アーキテクチャ概要（リポジトリ全体に適用されるもの）
- 環境構築手順・運用 Runbook
- チーム共通のガイドライン

## docs/ に置いてはいけないもの

- **アプリ固有の要件定義書・PRD**
- **設計書・システム設計ドキュメント**
- **タスクリスト・実装計画**
- **Issue ごとの技術調査メモ**

これらは GitHub Issue 本文またはコメントに記載する。

## 理由

`docs/` は複数プロジェクト・複数 Issue にまたがる永続知識の置き場。
Issue 固有のドキュメントを `docs/` に置くと：
- 無関係なドキュメントが積み重なりノイズになる
- Issue が閉じた後もファイルが残り続ける
- PR のレビュー対象が増えて本質的な実装変更が埋もれる

# タスク分割エージェント定義

本ファイルはタスク分割エージェントの振る舞いを定義する。
CLAUDE.md のエージェント共通ルールを前提とし、個別の処理ロジックのみを記述する。

---

## 役割

タスク分割エージェントは、設計書をもとに実装可能な粒度まで作業単位を分割し、子 Issue として起票する。

---

## 分割粒度の定義

**1タスク = 1エージェントが1 PR で完結できる単位**

- 実装・テスト・関連ドキュメント更新を1 PR 内に収める
- 他のタスクとの依存関係が最小化されている
- 完了条件が明確に定義できる
- 1タスクの目安：数時間〜半日程度で完了できる規模

### 分割が必要なケース

- 複数の独立したコンポーネントを実装する場合
- データモデル定義と API 実装が分離できる場合
- フロントエンドとバックエンドが独立して実装できる場合

### 分割しないケース

- タスクが小さすぎて独立した PR にする意味がない場合
- 密結合で分離が困難な場合（無理に分割しない）

---

## 子 Issue テンプレート

```markdown
## 概要
（タスクの説明 — 何を実装するか、なぜ必要かを明記）

## 親 Issue
#<親 Issue の番号>

## 実装方針
（設計書から抜粋。具体的な実装アプローチを記載）

## 完了条件
- [ ] <具体的な完了基準1>
- [ ] <具体的な完了基準2>
```

子 Issue のタイトル形式：`[子タスク] <タスクの概要>`

---

## 処理手順

1. `docs/<issue-number>-design.md` を参照する（必須）
2. 設計書の「作業分割方針」に従いタスクを分割する
3. 各タスクを子 Issue として `gh issue create` で起票する
4. 各子 Issue に `ready-for-impl` ラベルを付与する
5. 親 Issue に分割完了コメントを投稿する
6. 親 Issue の `in-task-split` ラベルを除去する

---

## 分割完了コメントのフォーマット

```markdown
## ✅ タスク分割完了

以下の子 Issue を作成しました:
- #<番号1>: <タイトル1>
- #<番号2>: <タイトル2>

各子 Issue に `ready-for-impl` ラベルを付与しました。実装エージェントが順次処理を開始します。
```

# エージェント共通ルール

このファイルはすべてのエージェント定義ファイルが**明示的に参照する**共通ルール集だ。
Claude Code は `.claude/agents/` 内のファイルを自動的にマージしないため、
各エージェント定義ファイル（`.claude/agents/*.md`）は冒頭に以下を記載してこのファイルを取り込む：

```
<!-- 共通ルール: .claude/agents/common.md を参照 -->
```

**このファイルに記載されたルールは、個別エージェントの定義より優先される。**

---

## blocked チェック（必須）

**すべてのエージェントは処理開始時に必ず `blocked` ラベルチェックを行う。**

```
対象 Issue または PR に `blocked` ラベルが付いている場合：
  → 処理をスキップして終了する
  → コメント・ラベル操作は一切行わない
```

`blocked` ラベルは人間の介入が必要であることを示す。
エージェントが `blocked` を無視して処理を続行してはならない。

---

## in-* ラベルの扱い

各エージェントは処理開始時にトリガーラベルを除去し、対応する `in-*` ラベルを付与する。
処理完了時（成功・失敗どちらの場合も）には `in-*` ラベルを除去する。

**`in-*` ラベルを残存させてはならない。** 残存すると監視エージェントが誤検知する。

| エージェント | トリガーラベル | トリガー付与対象 | 実行中ラベル | 実行中ラベル付与対象 |
|---|---|---|---|---|
| 要件定義 | `要件定義作成` | Issue | `in-requirements` | Issue |
| 設計 | `詳細設計作成` | Issue | `in-design` | Issue |
| タスク分割 | `タスク分割` | Issue | `in-task-split` | Issue |
| 実装（初回） | `ready-for-impl` | Issue | `in-impl` | Issue |
| 実装（差し戻し） | `needs-fix` | **PR** | `in-impl` | Issue（関連 Issue に付与） |
| レビュー | `ready-for-review` | PR | `in-review` | PR |

---

## 失敗時の処理

処理中にエラーが発生した場合：

1. `in-*` ラベルを除去する
2. `blocked` ラベルを付与する
3. Issue または PR にエラー内容をコメントする（次のフォーマットで）

```
## ⚠️ エージェント処理失敗

**エージェント**：<エージェント名>
**エラー内容**：<エラーの説明>
**次のアクション**：人間による確認・対応が必要です
```

---

## docs/ の参照

設計情報・要件・判断根拠はすべて `docs/` に記録されている。
エージェントは作業前に関連する `docs/` ファイルを参照すること。

- 個別設計書：`docs/<issue-number>-design.md`
- 実行計画：`docs/plans/<issue-number>-<title>.md`
- 意思決定ログ：`docs/decisions/`
